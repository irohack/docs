    gitコマンド覚え書き

    git init

    git init

    ローカルリポジトリの作成、初期化。

    git clone

    git clone <ssh>

    リモートリポジトリにあるデータをローカルリポジトリにまるまるダウンロードする。

    git pull

    git pull

    リモートリポジトリから前回のダウンロードからの変更内容を拾ってきて、データを最新版に更新する。 git cloneとは違い、こちらは頻繁に実行しないといけない。

    git branch

    ブランチの作成

    git branch <ブランチ名>

    ローカルブランチとリモート追跡ブランチの両方のブランチを表示

    git branch -a

    緑色 -> 現在いるブランチ 標準 -> ローカルブランチ 赤色 -> リモート追跡ブランチ

    ブランチの削除

    git branch -D <ブランチ名>

    git checkout

    git checkout <ブランチ名>

    git branch -aで確認した別ブランチに移動する。

    編集したファイルを素の状態に戻す

    git checkout .

    git status

    git status

    変更したファイルが赤色で表示される。git add後に見ると緑色で表示される。 git commit後には何も表示されなくなる。また、何もファイルを変更してない場合も何も表示されない。

    git add

    git add .

    変更したファイルすべてをcommit予定ボックスに追加する。

    git commit

    git commit -m "コメント"

    変更したファイルをローカルリポジトリにコミットする。 コメントのところにチケットの名前を入れるとそのチケットにコミット履歴として残る。

    git push

    git push origin <pushしたいブランチ名>

    ローカルリポジトリの変更内容をリモートリポジトリに反映する。 コミット時につけたコメントのチケットに、詳細がコメントされる

    git log

    git log

    commit履歴を確認できる。黄色で表示されているアルファベットと数字の列を用いて、そのコミット時までファイルを戻すこともできる。

    git logを用いて特定のファイルのみ前の状態に戻す

    git log <ファイルパス>
git checkout <英数字列> <ファイルパス>

    指定したファイルを以前のバージョンに戻す。 git logで取得した英数字列をgit checkoutに入れる。
